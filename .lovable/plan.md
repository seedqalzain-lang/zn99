## نظام إدارة الضمانات - تاج الملوك

سأبني نظاماً متكاملاً لإدارة ضمانات اللواصق (PPF/التظليل) مدمجاً في الموقع الحالي، باللغة العربية بالكامل مع RTL.

### المراحل

**المرحلة 1: قاعدة البيانات (Supabase)**

جداول جديدة:
- `customers` — العملاء (مرتبطين بـ auth.users): الاسم، الجوال، البريد، تاريخ التسجيل
- `branches` — الفروع: الاسم، العنوان، الهاتف
- `warranty_brands` — ماركات السيارات: الاسم، الشعار، الحالة
- `film_types` — أنواع اللاصق: الاسم، الماركة (اختياري)، مدة الضمان بالأشهر، الوصف
- `warranties` — الضمانات: رقم الضمان (فريد)، customer_id، brand_id، film_type_id، رقم الهيكل، branch_id، تاريخ التفعيل، تاريخ الانتهاء، الحالة (active/expired/cancelled)، ملاحظات، employee_id
- `user_roles` — الصلاحيات: user_id, role (admin/branch_staff/customer), branch_id

سياسات RLS كاملة + GRANTs، ودوال `has_role` و `generate_warranty_number`.

**المرحلة 2: المصادقة**
- صفحة `/warranty/auth` — تسجيل دخول/إنشاء حساب بالبريد + كلمة مرور (+ حقل جوال للعميل) واستعادة كلمة المرور
- استخدام Lovable Cloud auth، إنشاء profile تلقائي عبر trigger
- مسار محمي `_authenticated/` للوحة العميل

**المرحلة 3: واجهة العميل**
- `/warranty/activate` — تفعيل الضمان (نموذج كامل)
- `/warranty/dashboard` — قائمة ضمانات العميل مع البحث والتصفية والحالة
- `/warranty/certificate/$id` — شهادة رقمية بشعار تاج الملوك + QR Code + زر طباعة + تحميل PDF (jsPDF + html2canvas)

**المرحلة 4: التحقق العام**
- `/warranty/verify` — صفحة عامة لإدخال رقم الضمان أو مسح QR وعرض الحالة والتفاصيل الأساسية

**المرحلة 5: لوحة الإدارة**
- `/warranty/admin` — محمية بدور admin
  - تبويبات: لوحة الإحصائيات، الضمانات، العملاء، الماركات، أنواع اللاصق، الفروع، الموظفون
  - CRUD كامل مع البحث/الفرز/التصفية
  - تمديد/إلغاء/إعادة إصدار الضمان
  - رسوم بيانية (recharts) للإحصائيات

**المرحلة 6: تكامل التصميم**
- إضافة رابط "الضمانات" في الـ Header
- استخدام نفس نظام ألوان وخطوط تاج الملوك
- دعم الوضع الليلي/الفاتح (مفعّل أصلاً)
- متجاوب بالكامل

### التفاصيل التقنية

- **PDF**: `jspdf` + `html2canvas` لتحويل قالب الشهادة إلى PDF
- **QR Code**: `qrcode.react` يولّد رابط `/warranty/verify?n=<warranty_number>`
- **Server Functions**: `createServerFn` مع `requireSupabaseAuth` لجميع عمليات الإدارة
- **التحقق العام**: server fn بدون مصادقة يقرأ بيانات الضمان الأساسية فقط (سياسة anon SELECT محدودة بأعمدة آمنة)
- **رقم الضمان**: تنسيق `TM-YYYY-XXXXXX` يتولّد تلقائياً

### ملاحظات

- لن أحذف أو أعدّل المنتجات/الخدمات الحالية
- النظام سيكون في مسار `/warranty/*` منفصل عن المتجر
- المدير الأول يُضاف يدوياً عبر إدخال user_id في `user_roles` بعد تسجيل أول حساب

هل أبدأ التنفيذ؟ أو هل تريد تعديل أي جزء (مثلاً: تسجيل الدخول بالجوال OTP بدل البريد، أو تغيير الصلاحيات)؟
